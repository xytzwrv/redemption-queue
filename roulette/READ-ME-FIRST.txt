CHAT ROULETTE — OBS Overlay (fake points, real odds)
====================================================

WHAT THIS IS
  A European roulette your chat plays with !bet commands, using
  symbolic ("fake") points you control - so it pays REAL roulette odds
  (35:1 straight up, 2:1 dozens/columns, 1:1 red/black) on a full 0-36
  wheel. No Twitch login needed: it reads chat anonymously. Scores are
  tracked per Twitch USER-ID (so renames never lose a balance) and
  persist across OBS reloading the source. Same 1-bit field-system look
  as the rest of the set.

WHAT'S IN THIS FOLDER
  roulette.html      - the applet
  READ-ME-FIRST.txt  - this file
  (loads ../shared/field.js and ../shared/field.css)

OBS SETUP
  1. Start the local server (double-click start-windows.bat one folder up).
  2. Add a "Browser Source":
       - URL:    http://localhost:8777/roulette/roulette.html
       - Width:  1920    Height: 1080
  3. First run: type your Twitch channel name in the setup box (or add
     ?channel=yourname to the URL). It connects to your chat read-only -
     no login, no token, nothing to leak.
  4. Right-click -> Interact to use START ROUND / CLOSE BETS / AUTO /
     EXPORT / IMPORT (or press SPACE to start/close a round).

HOW CHAT PLAYS (viewers type in your chat)
  !bet red 100            1:1   (red / black / odd / even / low / high)
  !bet 17 50              35:1  (straight up on a single number 0-36)
  !bet dozen1 200         2:1   (dozen1=1-12, dozen2=13-24, dozen3=25-36;
                                 d1/d2/d3 also work)
  !bet col1 100           2:1   (columns; c1/c2/c3 also work)
  amount can be a number, "all", or "half".
  - Everyone starts with 1000 points; bets can be placed until you close
    the round; multiple bets allowed up to your balance.
  - Go bust and you're topped up to 100 so you can keep playing.
  - The green 0 is the house edge (loses all even-money/dozen/column
    bets) - exactly like a real table.

ROUND FLOW
  START ROUND -> BETS OPEN (countdown; chat bets) -> CLOSE BETS (or the
  timer) -> the wheel spins -> it lands -> everyone is paid, the TOP
  STACKS leaderboard and bet feed update. Turn AUTO on to loop rounds
  automatically. ?secs=45 sets the betting window; ?auto=1 auto-loops.

SCORES / PERSISTENCE (important)
  - Balances are keyed by Twitch user-id, so a viewer keeps their score
    even if they rename, and even while they're not in chat.
  - They persist across OBS reloading the browser source (saved in the
    browser's local storage, not just memory).
  - EXPORT downloads a JSON backup of all scores; IMPORT restores one.
    Do an EXPORT now and then - local storage lives in one browser cache,
    so an OBS "clear cache" or a new PC would otherwise lose it.
  - Want bullet-proof, cross-PC, public-leaderboard persistence? The
    storage lives in ONE place in the code - the `Store` object near the
    top of roulette.html. Swap its load/save/commit for fetch() calls to
    a tiny service on your VPS (or Supabase) and nothing else changes.

FEEDBACK IN CHAT (optional, later)
  Anonymous read can't post back, so bet confirmations + balances show
  on the OVERLAY (bet feed + TOP STACKS). If you want the game to reply
  in chat (e.g. answer !balance), that's the only part that needs a bot
  account token - a later add-on.

VISUALS CONSOLE (the "FX" panel, top-left; press H to hide)
    - PIXEL-PERFECT / ALPHA BG / PALETTE / RESET (RESET is visuals only -
      it does NOT touch scores).
  Keys: SPACE start/close round, E export scores, H console, F fullscreen.

NOTES
  - ?demo=1 runs a simulated chat of bettors so you can see it work with
    no real viewers (add &secs=15 for quick rounds).
  - The wheel pockets use real red/black/green so chat can read their
    bets; the rest of the UI follows the palette you pick.
  - Pixels render 1:1 for the crispest look; add ?px=2..12 for chunkier.
