CHAT BUBBLES - OBS Overlay
============================

WHAT THIS IS
  Every chat message spawns a white rectangle with black console-font
  text and a colored username, drops in with a "pop" sound, and floats
  upward - bouncing off all four edges of the screen, tumbling and
  tilting from hitting every other bubble on screen, in a real (if
  simple) 2D physics simulation, not just a CSS animation. After 20
  seconds the ceiling opens up for that bubble and it floats off the
  top of the screen for good instead of bouncing forever.

  No Twitch login needed - it reads chat anonymously (read-only), same
  as the map/POI selector and dice. Just type your channel name once.

  Emotes render inline as images, and Cheers (bit donations) get a
  colored glow around the bubble plus a small bits-count badge - see
  EMOTES & BITS below.

WHAT'S IN THIS FOLDER
  chatbubbles.html   - the overlay (the only important file)
  READ-ME-FIRST.txt  - this file

OBS SETUP
  1. Start the local server (double-click start-windows.bat one folder
     up, or `python3 -m http.server 8777` from the repo root).
  2. Add a "Browser Source":
       - URL:    http://localhost:8777/chatbubbles/chatbubbles.html
       - Width:  1920    Height: 1080
       - Make sure it isn't muted in OBS's Audio Mixer, or you won't
         hear the pop sound.
  3. Right-click -> Interact, type your Twitch channel name in the
     on-screen box, and click "Connect Chat". (One time - it's
     remembered in this browser source after that.) Or skip the box
     entirely by adding ?channel=yourname to the URL.
  4. That's it. Chat something to test it, or use ?demo=1 (below).

EMOTES & BITS
  - Twitch and channel-specific emotes render as inline images, pulled
    straight from Twitch's public emote CDN - no API key needed, still
    fully anonymous. Third-party emotes (BTTV/FFZ/7TV) aren't supported,
    since those aren't in the anonymous IRC tag data.
  - A Cheer (bit donation in chat) gets a colored glow around the whole
    bubble plus a small bits badge (e.g. "diamond 500"), colored by a
    rough tier: gray under 100, purple 100+, green 1000+, blue 5000+,
    red 10000+. This is a homage to Twitch's real Cheermote tier colors,
    not the official animated Cheermote graphics - those require an
    authenticated API call, which would mean asking viewers/the
    broadcaster to log in just to see them, breaking the no-login
    design of this overlay.

TIPS
  - Press H (after clicking the source once via Interact, so it has
    focus) to show a small mute/size panel - a speaker icon to mute the
    pop sound, and a slider for bubble size (up to 2x today's size).
    It's hidden by default and stays hidden on the actual stream output
    - OBS renders whatever's on the page regardless of Interact mode,
    so this only ever shows up while you're actively adjusting it.
    Both settings are remembered (localStorage) after that. Press H
    again to hide the panel. You can also set them directly in the URL:
    ?mute=1 and ?size=1.5 (0.5-2). The size slider only affects bubbles
    spawned from then on - it doesn't resize ones already on screen.
  - Add  ?demo=1  to the URL to preview with simulated chat, no real
    channel needed:
        http://localhost:8777/chatbubbles/chatbubbles.html?demo=1
  - Known bots (Nightbot, StreamElements, etc.) are filtered out by
    default - edit CONFIG.ignore in chatbubbles.html to add more.
  - One chatter can't flood the screen: there's a per-user cooldown
    (CONFIG.perUserCooldownMs, default 4s) and a hard cap on concurrent
    bubbles (CONFIG.maxBubbles, default 60) - past the cap, the oldest
    bubble fades out early to make room for the new one.
  - Long messages are truncated (CONFIG.maxMessageChars, default 200)
    so a giant paste can't spawn a screen-dominating rectangle.
  - Tune the feel in CONFIG at the top of the <script> block:
    lifespanMs (20000, time before the ceiling opens), buoyancy/
    driftJitter/maxSpeed (how floaty/fast they are), wallRestitution/
    bubbleRestitution (how bouncy), angularDrag/spinFromHit/
    maxAngularSpeed (how much they tumble from hits), and
    sound.volume/startFreq/endFreq/duration (the pop sound).
  - The pop sound is synthesized (a short Web Audio oscillator blip),
    not an audio file - nothing to download or go missing. Some
    browsers block audio until the page has had a click/keypress; OBS
    Browser Sources are normally exempt from that, but if you're
    previewing in a regular browser tab and don't hear anything, click
    the page once.
